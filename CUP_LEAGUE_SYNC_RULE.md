# PGM联赛开发规则 - 杯赛与联赛同步

**生效日期**: 2026-04-20  
**版本**: v1.0.162+  
**状态**: 永久有效

---

## 核心规则

**杯赛的所有页面布局、样式、交互与联赛保持完全一致。**

这意味着：
1. 任何联赛的UI修改，杯赛必须同步修改
2. 杯赛不会单独设计不同的展示方式
3. 更新日志中不再单独说明杯赛同步的情况

## 技术实现要点

### 1. 联赛 vs 杯赛代码位置
- **联赛**: `renderSchedule()` 函数 (~line 2044)
- **杯赛**: `renderCupSchedule()` 函数 (~line 1408)

### 2. 区分联赛和杯赛的特征
```javascript
// 联赛特征
const isLeague = !!home.league;  // home.league 存在

// 杯赛特征  
const isCup = !!CUP_NAMES[league];  // CUP_NAMES 中包含
const canEnterCup = state.currentUser && 
                    state.currentUser.recorderType === 'cup' && 
                    state.currentUser.cupId === cupId;
```

### 3. 同步检查清单
修改联赛代码时，必须同时检查杯赛：
- [ ] 轮次切换栏样式
- [ ] 比赛卡片样式
- [ ] 比分区域样式
- [ ] 录入按钮样式
- [ ] 状态标签样式
- [ ] 球队详情页展示
- [ ] 更新日志页面
- [ ] 任何新增交互效果

---

## 例外情况

以下情况杯赛可能有特殊处理：
1. **数据结构不同**（杯赛有 `cupId`，联赛有 `league`）
2. **权限判断不同**（`canEnterCup` vs `canEnterMatch`）
3. **轮次逻辑不同**（杯赛可能有淘汰赛轮次命名）

---

## 责任

**开发者职责**: 每次修改联赛UI时，主动检查并同步杯赛代码。  
**审查职责**: 代码审查时确认杯赛同步是否完成。
